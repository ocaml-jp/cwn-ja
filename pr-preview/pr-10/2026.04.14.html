YAMLx: pure-OCaml YAML 1.2/1.1 ライブラリ

Archive: https://discuss.ocaml.org/t/ann-yamlx-pure-ocaml-yaml-1-2-1-1-library/17970/1

Martin Jambon さんによるアナウンス

こんにちは、

YAML 1.2 および 1.1 標準の完全な実装である YAMLx の公開をお知らせできることを嬉しく思います。新しいアプリケーションでの設定ファイル形式としての使用や、他のアプリケーションの YAML ファイルの解析を含む、YAML のほとんどの用途をサポートすることを目的としています。

詳しい説明は リリースノート をご覧ください。

opam install yamlx で始めてください。次に yamlx コンパニオンコマンドを使って、お気に入りの YAML ファイルが正しく解析・解釈されるか確認してください。

以下は、ライブラリを使って YAML を JSON に変換する例です： 

(* Read YAML from stdin with YAMLx, print JSON to stdout with Yojson

   This is a demo showing how to use the YAMLx library with the default
   settings. 'YAMLx.Values.one_of_yaml_exn' offers options to restrict
   the input language or to force the interpretation as YAML 1.1 instead of
   YAML 1.2.

   Compile with:
     ocamlfind opt -o yaml-to-json \
       -package yamlx,yojson -linkpkg yaml_to_json.ml
*)

open Printf

let rec yojson_of_yamlx (x : YAMLx.value) : Yojson.Safe.t =
  match x with
  | Null _ -> `Null
  | Bool (_, x) -> `Bool x
  | Int (_, x) -> `Intlit (Int64.to_string x)
  | Float (_, x) -> `Float x
  | String (_, x) -> `String x
  | Seq (_, xs) -> `List (List.map yojson_of_yamlx xs)
  | Map (_, xs) ->
      `Assoc (List.map (fun (loc, k, v) ->
        match (k : YAMLx.value) with
        | String (_, k) -> (k, yojson_of_yamlx v)
        | _ ->
            ksprintf failwith "%s: only string keys are supported"
              (YAMLx.default_format_loc loc)
      ) xs)

let () =
  YAMLx.register_exception_printers ();
  stdin
  |> In_channel.input_all
  |> YAMLx.Values.one_of_yaml_exn
  |> yojson_of_yamlx
  |> Yojson.Safe.pretty_to_channel stdout
  |> print_newline

資金調達について

YAMLx は現在 AGPL ライセンスのもとで公開されています。現在資金調達キャンペーンを実施しており、目標額に達した時点で、ライセンスは全員に対してパーミッシブな ISC ライセンスへ移行されます。一定の金額以上を寄付された方には、即時に 商用ライセンス が付与されます。詳細は GitHub Sponsors のプロフィール をご覧ください。

ocaml-letsencrypt: OCaml で TLS 証明書を取得する方法

Archive: https://discuss.ocaml.org/t/ann-ocaml-letsencrypt-an-ocaml-way-to-get-tls-certificates/17975/1

Calascibetta Romain さんによるアナウンス

ocaml-letsencrypt（バージョン 2.1.0）のリリースをお知らせできることを嬉しく思います。このバージョンでは、スケジューラや HTTP クライアントとは独立して動作できる新しい API が導入されています。また、新しい jws ライブラリを使った書き直しでもあり、jsont ライブラリ（yojson の代わりに）を利用しています。

ocaml-letsencrypt は、DNS、HTTP、ACME-TLS の3つの方式で証明書を取得するために必要な チャレンジ を実装しています。これにより、（例えば Vif を使って実装された）ウェブサイトのデプロイ戦略を完全に OCaml で提供できるようになります。

私たちのコープでは、以下のようなプロジェクトで使用しています：

  • dns-letsencrypt-secondary：セカンダリ DNS サーバーとして動作し、特定のドメインに対して DNS チャレンジを実行して証明書と秘密鍵を生成するユニカーネル。通常、プライマリ DNS サーバーである primary-git と連携して動作します。
  • contruno：TLS リバースプロキシとして動作し、HTTP チャレンジを実行できるユニカーネル。現在、mnet を使って完全に書き直し中です。

ウェブサイトのデプロイがうまくいくことを願っています。Happy hacking! jws もまた OCaml エコシステムの中でライブラリとして定着してくれることを願っています。

I18n 5.0 — あらゆる OCaml アプリの国際化を簡単に

Archive: https://discuss.ocaml.org/t/ann-i18n-5-0-internationalisation-made-easy-for-any-ocaml-app/17981/1

Vincent Balat さんによるアナウンス

Ocsigen チームは、OCaml 向けの小さながら実用的な国際化ライブラリ ocsigen-i18n 5.0 のリリースをお知らせします。

このリリースの大きな変更点：ocsigen-i18n は Eliom への依存がなくなりました。あらゆる OCaml プロジェクト で使用できるようになり、必要に応じて Tyxml および Eliom 向けのオプション拡張が利用できます。

ocsigen-i18n は元々 Be Sport の @sagotch によって書かれました。このリリースは Habib の作業によって実現し、IRILL の支援を受けています。

opam install ocsigen-i18n

使い方

翻訳はシンプルな TSV ファイル（1行に1キー、1列に1言語）で記述します： 

foo                 This is a simple key.        Ceci est une clé toute simple.
a_human             a human                      un humain
bar                 I am {{x}}.                  Je suis {{x}}.
baz                 There {{{c?are||is an}}} apple{{{c?s||}}} here!  Il y a {{{c?des||une}}} pomme{{{c?s||}}} ici !
bu                  I am {{x %s}} ({{n %d}}).    Je suis {{x %s}} ({{n %d}}).

ミニテンプレート言語は以下をサポートしています：

  • {{x}} 文字列変数 ~x
  • {{x %d}} 指定されたフォーマット指定子を使った型付き変数
  • {{{c?yes||no}}} 2つの文字列を切り替えるオプションの真偽値 ?c

コード内では、PPX 拡張が [%i18n key] を適切な呼び出しに変換します： 

print_endline [%i18n foo];
print_endline [%i18n bar ~x:[%i18n a_human]];
print_endline [%i18n baz ~c:(nb > 1)];
print_endline [%i18n bu ~x:"Jean-Michel" ~n:42];

(* Switching language explicitly *)
print_endline [%i18n foo ~lang:My_i18n.Fr];

現在の言語は、スワップ可能なミュータブル参照に保持されます（セッション/タブごとの言語が必要な場合は Eliom のスコープ付き参照に置き換えることもできます）。

Dune との統合

このツールは Dune に非常に自然に組み込めます。以下のルールで TSV ファイルから OCaml モジュールを生成します： 

(rule
 (target example_i18n.ml)
 (deps example_i18n.tsv)
 (action
  (run %{bin:ocsigen-i18n} --languages en,fr --input-file %{deps}
                           --output-file %{target})))

ライブラリ/実行ファイルに PPX を組み込みます： 

(preprocess (pps ocsigen-i18n -- --default-module Example_i18n))

純粋な OCaml プロジェクトに必要なのはこれだけです。

Tyxml サポート

ジェネレーターに --tyxml を渡すと、同じ [%i18n key] 式が文字列の代わりに Tyxml HTML 要素のリスト を生成するようになります： 

(* Builds an HTML fragment, ready to drop into a Tyxml tree *)
let greeting = [%i18n bar ~x:[%i18n a_human]]

変数自体を HTML ノードのリストにできるため、翻訳テキストとマークアップを自然に混在させることができます： 

[%i18n bar ~x:[ txt "Jean-Michel ("
              ; txt (string_of_int id)
              ; txt ")" ]]

Tyxml モードでプレーン文字列が必要な場合（属性など）は、S. プレフィックスを付けてください： 

[%i18n S.bar ~x:[%i18n S.a_human]]   (* string output *)

Eliom サポート

クライアント–サーバー Eliom アプリの場合は --eliom を渡します。ジェネレーターは .eliom ファイルを生成し（両側で同じ翻訳が利用可能になります）、--tyxml を含意し、さらに言語型に [@@deriving json] を追加してワイヤー越しにシリアライズできるようにします： 

(rule
 (target example_i18n.eliom)
 (deps example_i18n.tsv)
 (action
  (run %{bin:ocsigen-i18n} --eliom --languages en,fr --input-file %{deps}
                           --output-file %{target})))

複数の TSV ファイル

翻訳を複数のファイルに分割することができます。PPX はモジュールパスを使って適切なファイルを見つけます： 

[%i18n foo]                          (* default module *)
[%i18n MyI18n.foo]                   (* MyI18n.foo *)
[%i18n MyI18n.S.bar ~x:[%i18n S.foo]]

オプションの --prefix / --suffix フラグを使うと、呼び出し側を簡潔に保てます（[%i18n Feature.foo] → Pr_Feature_i18n.foo ()）。

Happy translating!

ATD 4.1.0 - YAML サポートが追加されました

Archive: https://discuss.ocaml.org/t/ann-atd-4-1-0-now-with-yaml-support/17982/1

Martin Jambon さんによるアナウンス

ATD に、YAML ファイルや他のフォーマットからのデータを最小限の手間で便利に読み込む OCaml サポートが追加されました。具体的には：

  1. 設定データの型を定義 します。
  2. atdml --mode jsonlike app_config.atd が、ソースの場所情報を含む JSON ライクな AST を設定データ構造に変換する OCaml コードを生成します。
  3. プログラム が YAML ファイルを読み込み、JSON ライクなツリーに変換し、生成された関数を呼び出して OCaml データ構造に変換します。
  4. 設定が不正またはが不完全な場合、アプリケーションは有用なエラーメッセージを出力します。

例 がディストリビューションに含まれています。最近リリースされた YAMLx ライブラリ を使用しています。

YAMLx にはこれまでの作業に対する資金提供者と、パーミッシブな ISC ライセンス を全員に向けて解放するためのスポンサーが必要です。または、アプリケーションのソースコードをユーザーと共有することに問題がなければ、AGPL ライセンス の条件を受け入れることもできます。

ソースの場所情報が利用可能であれば、他の入力ファイル形式や他の YAML ライブラリも最小限の手間で同様に使用できます。そのためには、AST を JSON ライクな AST に変換する ブリッジ が必要になります。
