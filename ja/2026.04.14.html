YAMLx：純粋 OCaml 製の YAML 1.2/1.1 ライブラリ

Archive: https://discuss.ocaml.org/t/ann-yamlx-pure-ocaml-yaml-1-2-1-1-library/17970/1

Martin Jambon さんによる告知

こんにちは、

YAML 1.2 および 1.1 標準の完全な実装である YAMLx の公開を嬉しくお知らせします。新しいアプリケーションの設定ファイル形式としての利用や、他のアプリケーションの YAML ファイルの解析など、YAML の大部分のユースケースをサポートすることを目的としています。

詳細については リリースノート をご覧ください。

opam install yamlx で始められます。インストール後、yamlx コンパニオンコマンドを使って、お気に入りの YAML ファイルが正しく解析・解釈されるかを確認してください。

以下は YAML を JSON に変換するためのライブラリの使用例です： 

(* Read YAML from stdin with YAMLx, print JSON to stdout with Yojson

   This is a demo showing how to use the YAMLx library with the default
   settings. 'YAMLx.Values.one_of_yaml_exn' offers options to restrict
   the input language or to force the interpretation as YAML 1.1 instead of
   YAML 1.2.

   Compile with:
     ocamlfind opt -o yaml-to-json \
       -package yamlx,yojson -linkpkg yaml_to_json.ml
*)

open Printf

let rec yojson_of_yamlx (x : YAMLx.value) : Yojson.Safe.t =
  match x with
  | Null _ -> `Null
  | Bool (_, x) -> `Bool x
  | Int (_, x) -> `Intlit (Int64.to_string x)
  | Float (_, x) -> `Float x
  | String (_, x) -> `String x
  | Seq (_, xs) -> `List (List.map yojson_of_yamlx xs)
  | Map (_, xs) ->
      `Assoc (List.map (fun (loc, k, v) ->
        match (k : YAMLx.value) with
        | String (_, k) -> (k, yojson_of_yamlx v)
        | _ ->
            ksprintf failwith "%s: only string keys are supported"
              (YAMLx.default_format_loc loc)
      ) xs)

let () =
  YAMLx.register_exception_printers ();
  stdin
  |> In_channel.input_all
  |> YAMLx.Values.one_of_yaml_exn
  |> yojson_of_yamlx
  |> Yojson.Safe.pretty_to_channel stdout
  |> print_newline

資金調達について

YAMLx は現在 AGPL のもとで公開されています。現在クラウドファンディングを実施中で、目標額に達した時点でライセンスが全ての人に対して寛容な ISC ライセンスに切り替わります。一定額以上のご支援者には即座に 商用ライセンス が付与されます。詳細は GitHub Sponsors プロフィール をご覧ください。

ocaml-letsencrypt：TLS 証明書を取得するための OCaml の方法

Archive: https://discuss.ocaml.org/t/ann-ocaml-letsencrypt-an-ocaml-way-to-get-tls-certificates/17975/1

Calascibetta Romain さんによる告知

ocaml-letsencrypt（バージョン 2.1.0）のリリースをお知らせできることを嬉しく思います。このバージョンでは、スケジューラや HTTP クライアントに依存せずに動作できる新しい API が導入されました。また、jsont ライブラリ（yojson の代わりに）を利用する新しい jws ライブラリを使った書き直しでもあります。

ocaml-letsencrypt は、DNS、HTTP、ACME-TLS の三つの方法で証明書を取得するために必要な チャレンジ を実装しています。そのため、（例えば Vif を使って実装された）ウェブサイトのデプロイ戦略を完全に OCaml で提供することが可能です。

私たちのコープでは以下のようなプロジェクトで使用しています：

  • dns-letsencrypt-secondary：セカンダリ DNS サーバーとして動作し、特定のドメインの DNS チャレンジを実行して証明書と秘密鍵を生成できる unikernel。通常、プライマリ DNS サーバーである primary-git と連携して動作します。
  • contruno：TLS リバースプロキシとして動作し、HTTP チャレンジを実行できる unikernel。現在 mnet を使って完全に書き直し中です。

ウェブサイトのデプロイがうまくいくことを願っています。Happy hacking！jws も OCaml エコシステムの中でライブラリとしての地位を確立できることを期待しています。

I18n 5.0 — あらゆる OCaml アプリに簡単な国際化を

Archive: https://discuss.ocaml.org/t/ann-i18n-5-0-internationalisation-made-easy-for-any-ocaml-app/17981/1

Vincent Balat さんによる告知

Ocsigen チームは ocsigen-i18n 5.0 のリリースをお知らせします。OCaml 向けの小さくも実用的な国際化ライブラリです。

今回のリリースでの大きな変更点：ocsigen-i18n が Eliom に縛られなくなりました。必要に応じて Tyxml や Eliom 向けのオプション拡張を使いながら、あらゆる OCaml プロジェクト で使用できるようになりました。

ocsigen-i18n はもともと Be Sport の @sagotch によって書かれました。今回のリリースは Habib の作業によって実現し、IRILL から資金提供を受けています。

opam install ocsigen-i18n

仕組み

翻訳は平易な TSV ファイル（1行1キー、1列1言語）に記述します： 

foo                 This is a simple key.        Ceci est une clé toute simple.
a_human             a human                      un humain
bar                 I am {{x}}.                  Je suis {{x}}.
baz                 There {{{c?are||is an}}} apple{{{c?s||}}} here!  Il y a {{{c?des||une}}} pomme{{{c?s||}}} ici !
bu                  I am {{x %s}} ({{n %d}}).    Je suis {{x %s}} ({{n %d}}).

ミニテンプレート言語は以下をサポートしています：

  • {{x}} 文字列変数 x
  • {{x %d}} 指定したフォーマット指定子を使う型付き変数
  • {{{c?yes||no}}} 二つの文字列を切り替えるオプションのブール値 ?c

コード中では PPX 拡張 が [%i18n key] を適切な呼び出しに変換します： 

print_endline [%i18n foo];
print_endline [%i18n bar ~x:[%i18n a_human]];
print_endline [%i18n baz ~c:(nb > 1)];
print_endline [%i18n bu ~x:"Jean-Michel" ~n:42];

(* 言語を明示的に切り替える *)
print_endline [%i18n foo ~lang:My_i18n.Fr];

現在の言語は、スワップ可能なミュータブル参照に保持されます（セッション/タブごとの言語が必要な場合は Eliom スコープ参照に置き換えることもできます）。

Dune との統合

このツールは Dune と非常に自然に統合できます。TSV ファイルから OCaml モジュールを生成するルールを記述します： 

(rule
 (target example_i18n.ml)
 (deps example_i18n.tsv)
 (action
  (run %{bin:ocsigen-i18n} --languages en,fr --input-file %{deps}
                           --output-file %{target})))

そして、ライブラリ/実行ファイルに PPX を組み込みます： 

(preprocess (pps ocsigen-i18n -- --default-module Example_i18n))

純粋な OCaml プロジェクトにはこれだけで十分です。

Tyxml サポート

ジェネレータに --tyxml を渡すと、同じ [%i18n key] 式が文字列ではなく Tyxml HTML 要素のリスト を生成するようになります： 

(* Tyxml ツリーにそのまま挿入できる HTML フラグメントを構築する *)
let greeting = [%i18n bar ~x:[%i18n a_human]]

変数自体も HTML ノードのリストになれるため、翻訳されたテキストとマークアップを自然に組み合わせることができます： 

[%i18n bar ~x:[ txt "Jean-Michel ("
              ; txt (string_of_int id)
              ; txt ")" ]]

Tyxml モード中で（属性などに）プレーンな文字列が必要な場合は、先頭に S. を付けるだけです： 

[%i18n S.bar ~x:[%i18n S.a_human]]   (* 文字列出力 *)

Eliom サポート

クライアント・サーバー型の Eliom アプリには --eliom を渡します。ジェネレータは .eliom ファイル（両側で同じ翻訳が利用可能になる）を出力し、--tyxml が暗黙的に有効になり、ネットワーク越しにシリアライズできるよう言語型に [@@deriving json] が追加されます： 

(rule
 (target example_i18n.eliom)
 (deps example_i18n.tsv)
 (action
  (run %{bin:ocsigen-i18n} --eliom --languages en,fr --input-file %{deps}
                           --output-file %{target})))

複数の TSV ファイル

翻訳を複数のファイルに分割することができます。PPX はモジュールパスを使って適切なものを見つけます： 

[%i18n foo]                          (* デフォルトモジュール *)
[%i18n MyI18n.foo]                   (* MyI18n.foo *)
[%i18n MyI18n.S.bar ~x:[%i18n S.foo]]

オプションの --prefix / --suffix フラグで呼び出し箇所を簡潔に保てます（[%i18n Feature.foo] → Pr_Feature_i18n.foo ()）。

ATD 4.1.0 — YAML サポートが追加されました

Archive: https://discuss.ocaml.org/t/ann-atd-4-1-0-now-with-yaml-support/17982/1

Martin Jambon さんによる告知

ATD が YAML ファイルや他のフォーマットからデータを簡単に読み込む OCaml サポートを追加しました。具体的には：

  1. 設定データの型を定義します。
  2. atdml --mode jsonlike app_config.atd により、ソース位置を含む JSON ライクな AST を設定データ構造に変換する OCaml コードが生成されます。
  3. プログラム が YAML ファイルを読み込み、JSON ライクなツリーに変換してから、生成された関数を呼び出して OCaml データ構造に変換します。
  4. 設定が不正または不完全な場合、アプリケーションが有用なエラーメッセージを出力します。

サンプル がディストリビューションに含まれています。最近リリースされた YAMLx ライブラリ を使用しています。

YAMLx にはこれまでの作業に資金を提供し、寛容な ISC ライセンスを全員にアンロックする ための寛大なスポンサーが必要です。あるいは、アプリケーションのソースコードをユーザーと共有することに問題がなければ AGPL ライセンス の条件を受け入れることもできます。

ソース位置が利用可能であれば、他の入力ファイル形式や他の YAML ライブラリも最小限の労力で同様に使用できます。そのためには、自分の AST を JSON ライクな AST に変換する ブリッジ が必要です。
