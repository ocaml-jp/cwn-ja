* YAMLx: 純粋 OCaml 製 YAML 1.2/1.1 ライブラリ
:PROPERTIES:
:CUSTOM_ID: 3
:END:
Archive: https://discuss.ocaml.org/t/ann-yamlx-pure-ocaml-yaml-1-2-1-1-library/17970/1

** Martin Jambon による告知


こんにちは、

YAML 1.2 および 1.1 標準の完全な実装である [[https://github.com/mjambon/yamlx][YAMLx]] の公開をお知らせできることを嬉しく思います。このライブラリは、新しいアプリケーションの設定ファイル形式としての利用や、他のアプリケーションの YAML ファイルの解析など、YAML のほとんどのユースケースをサポートするために設計されています。

詳しい説明については、[[https://github.com/mjambon/yamlx/releases/tag/0.1.0][リリースノート]] をご覧ください。

~opam install yamlx~ でインストールできます。その後、~yamlx~ コンパニオンコマンドを使って、お気に入りの YAML ファイルが正しく解析・解釈されることを確認してください。

以下は YAML を JSON に変換するためのライブラリ使用例です：

#+begin_src ocaml
(* Read YAML from stdin with YAMLx, print JSON to stdout with Yojson

   This is a demo showing how to use the YAMLx library with the default
   settings. 'YAMLx.Values.one_of_yaml_exn' offers options to restrict
   the input language or to force the interpretation as YAML 1.1 instead of
   YAML 1.2.

   Compile with:
     ocamlfind opt -o yaml-to-json \
       -package yamlx,yojson -linkpkg yaml_to_json.ml
*)

open Printf

let rec yojson_of_yamlx (x : YAMLx.value) : Yojson.Safe.t =
  match x with
  | Null _ -> `Null
  | Bool (_, x) -> `Bool x
  | Int (_, x) -> `Intlit (Int64.to_string x)
  | Float (_, x) -> `Float x
  | String (_, x) -> `String x
  | Seq (_, xs) -> `List (List.map yojson_of_yamlx xs)
  | Map (_, xs) ->
      `Assoc (List.map (fun (loc, k, v) ->
        match (k : YAMLx.value) with
        | String (_, k) -> (k, yojson_of_yamlx v)
        | _ ->
            ksprintf failwith "%s: only string keys are supported"
              (YAMLx.default_format_loc loc)
      ) xs)

let () =
  YAMLx.register_exception_printers ();
  stdin
  |> In_channel.input_all
  |> YAMLx.Values.one_of_yaml_exn
  |> yojson_of_yamlx
  |> Yojson.Safe.pretty_to_channel stdout
  |> print_newline
#+end_src

* ocaml-letsencrypt — OCaml で TLS 証明書を取得する方法
:PROPERTIES:
:CUSTOM_ID: 4
:END:
Archive: https://discuss.ocaml.org/t/ann-ocaml-letsencrypt-an-ocaml-way-to-get-tls-certificates/17975/1

** Calascibetta Romain による告知


[[https://github.com/robur-coop/ocaml-letsencrypt][ocaml-letsencrypt]]（バージョン 2.1.0）のリリースをお知らせできることを嬉しく思います。このバージョンでは、スケジューラや HTTP クライアントに依存せずに動作できる新しい API が導入されました。また、新しい [[https://github.com/robur-coop/jws][jws]] ライブラリを用いた書き直しが行われ、[[https://github.com/ocaml-community/yojson][yojson]] の代わりに [[https://erratique.ch/logiciel/jsont][jsont]] ライブラリを使用するようになりました。

~ocaml-letsencrypt~ は、DNS、HTTP、ACME-TLS の3つの方式による証明書取得に必要な [[https://letsencrypt.org/fr/docs/challenge-types/][チャレンジ]] を実装しています。これにより、（例えば [[https://github.com/robur-coop/vif][Vif]] を使って実装された）ウェブサイトのデプロイ戦略を完全に OCaml で提供することができます。

私たちのコープでは以下のプロジェクトで使用しています：
- [[https://github.com/robur-coop/dns-letsencrypt-secondary/][~dns-letsencrypt-secondary~]]：特定のドメインに対して DNS チャレンジを実行し、証明書と秘密鍵を生成できるセカンダリ DNS サーバとして機能するユニカーネル。通常は [[https://github.com/robur-coop/dns-primary-git][~primary-git~]]（プライマリ DNS サーバ）と連携して動作します。
- [[https://github.com/dinosaure/contruno][~contruno~]]：HTTP チャレンジを実行できる TLS リバースプロキシとして機能するユニカーネル。現在 [[https://discuss.ocaml.org/t/ann-mnet-a-new-tcp-ip-stack-for-unikernels-in-ocaml/17851][~mnet~]] を使って完全に書き直し中です。

ウェブサイトのデプロイがうまくいくことを願っています。Happy hacking! ~jws~ も OCaml エコシステムの中でライブラリとして定着することを期待しています。

* I18n 5.0 — あらゆる OCaml アプリに簡単な国際化対応を
:PROPERTIES:
:CUSTOM_ID: 7
:END:
Archive: https://discuss.ocaml.org/t/ann-i18n-5-0-internationalisation-made-easy-for-any-ocaml-app/17981/1

** Vincent Balat による告知


Ocsigen チームは、OCaml 向けの小さくも実用的な国際化ライブラリ *ocsigen-i18n 5.0* のリリースをお知らせします。

このリリースの大きな変更点：ocsigen-i18n は Eliom に縛られなくなりました。*あらゆる OCaml プロジェクト* で使用できるようになり、必要に応じて Tyxml および Eliom 向けのオプション拡張を利用できます。

ocsigen-i18n は元々 Be Sport の @sagotch によって書かれました。このリリースは [[https://github.com/mhjd][Habib]] の作業によって実現され、[[https://www.irill.org][IRILL]] の資金援助を受けています。

#+begin_quote
opam install ocsigen-i18n
#+end_quote

*** 仕組み

翻訳は *TSV ファイル*（1行1キー、1列1言語）で記述します：

#+begin_example
foo                 This is a simple key.        Ceci est une clé toute simple.
a_human             a human                      un humain
bar                 I am {{x}}.                  Je suis {{x}}.
baz                 There {{{c?are||is an}}} apple{{{c?s||}}} here!  Il y a {{{c?des||une}}} pomme{{{c?s||}}} ici !
bu                  I am {{x %s}} ({{n %d}}).    Je suis {{x %s}} ({{n %d}}).
#+end_example

ミニテンプレート言語は以下をサポートします：

- ~{{x}}~ 文字列変数 ~~x~
- ~{{x %d}}~ 指定されたフォーマット指定子を使った型付き変数
- ~{{{c?yes||no}}}~ 2つの文字列を切り替えるオプションの真偽値 ~?c~

コード中では、*PPX 拡張* が ~[%i18n key]~ を適切な呼び出しに変換します：

#+begin_src ocaml
print_endline [%i18n foo];
print_endline [%i18n bar ~x:[%i18n a_human]];
print_endline [%i18n baz ~c:(nb > 1)];
print_endline [%i18n bu ~x:"Jean-Michel" ~n:42];

(* 言語を明示的に切り替える *)
print_endline [%i18n foo ~lang:My_i18n.Fr];
#+end_src

現在の言語はミュータブルな参照で保持されており、スワップ可能です（セッション/タブ単位の言語設定が必要な場合は Eliom スコープ付き参照に置き換えることもできます）。

*** Dune との統合

このツールは Dune と非常に自然に統合できます。ルールを使って TSV ファイルから OCaml モジュールを生成します：

#+begin_src lisp
(rule
 (target example_i18n.ml)
 (deps example_i18n.tsv)
 (action
  (run %{bin:ocsigen-i18n} --languages en,fr --input-file %{deps}
                           --output-file %{target})))
#+end_src

ライブラリ/実行ファイルに PPX を組み込みます：

#+begin_src lisp
(preprocess (pps ocsigen-i18n -- --default-module Example_i18n))
#+end_src

純粋な OCaml プロジェクトに必要なのはこれだけです。

*** Tyxml サポート

ジェネレータに ~--tyxml~ を渡すと、同じ ~[%i18n key]~ 式が文字列の代わりに *Tyxml HTML 要素のリスト* を生成します：

#+begin_src ocaml
(* HTML フラグメントを構築し、Tyxml ツリーにそのまま挿入できる *)
let greeting = [%i18n bar ~x:[%i18n a_human]]
#+end_src

変数自体も HTML ノードのリストにできるため、翻訳テキストとマークアップを自然に混在させることができます：

#+begin_src ocaml
[%i18n bar ~x:[ txt "Jean-Michel ("
              ; txt (string_of_int id)
              ; txt ")" ]]
#+end_src

Tyxml モードで（属性などに）プレーンな文字列が必要な場合は、~S.~ プレフィックスを付けるだけです：

#+begin_src ocaml
[%i18n S.bar ~x:[%i18n S.a_human]]   (* 文字列出力 *)
#+end_src

*** Eliom サポート

クライアント・サーバー Eliom アプリの場合は ~--eliom~ を渡します。ジェネレータは ~.eliom~ ファイルを出力し（両サイドで同じ翻訳が利用可能になります）、~--tyxml~ を暗黙的に有効にし、ワイヤを通してシリアライズできるよう言語型に ~[@@deriving json]~ を付加します：

#+begin_src lisp
(rule
 (target example_i18n.eliom)
 (deps example_i18n.tsv)
 (action
  (run %{bin:ocsigen-i18n} --eliom --languages en,fr --input-file %{deps}
                           --output-file %{target})))
#+end_src

*** 複数の TSV ファイル

翻訳を複数のファイルに分割できます。PPX はモジュールパスを使って適切なファイルを見つけます：

#+begin_src ocaml
[%i18n foo]                          (* デフォルトモジュール *)
[%i18n MyI18n.foo]                   (* MyI18n.foo *)
[%i18n MyI18n.S.bar ~x:[%i18n S.foo]]
#+end_src

オプションの ~--prefix~ / ~--suffix~ フラグを使うと、呼び出し箇所を簡潔に保てます（~[%i18n Feature.foo]~ → ~Pr_Feature_i18n.foo ()~）。

Happy translating!
      



* ATD 4.1.0 — YAML サポートが追加されました
:PROPERTIES:
:CUSTOM_ID: 8
:END:
Archive: https://discuss.ocaml.org/t/ann-atd-4-1-0-now-with-yaml-support/17982/1

** Martin Jambon による告知


[[https://github.com/ahrefs/atd][ATD]] に、YAML ファイルをはじめとするさまざまなフォーマットから最小限の手間でデータを読み込む OCaml サポートが追加されました。具体的には：

1. [[https://github.com/ahrefs/atd/blob/master/atd-yamlx/examples/app_config.atd][設定データの型を定義します]]。
2. ~atdml --mode jsonlike app_config.atd~ を実行すると、ソース位置情報を含む JSON ライクな AST を設定データ構造に変換する OCaml コードが生成されます。
3. [[https://github.com/ahrefs/atd/blob/master/atd-yamlx/examples/main.ml][プログラム]] が YAML ファイルを読み込み、JSON ライクなツリーに変換し、生成された関数を呼び出して OCaml データ構造に変換します。
4. 設定が不正または不完全な場合に、アプリケーションが有用なエラーメッセージを表示します。

[[https://github.com/ahrefs/atd/blob/master/atd-yamlx/examples][サンプル]] がディストリビューションに含まれています。[[https://discuss.ocaml.org/t/ann-yamlx-pure-ocaml-yaml-1-2-1-1-library/17970][最近リリースされた YAMLx ライブラリ]] を使用しています。

_なお、YAMLx がこれまでの作業資金を賄い [[https://github.com/mjambon/yamlx/blob/main/FUNDING.md][全員向けに寛容な ISC ライセンスに移行する]] ためには、寛大なスポンサーが必要です。または、アプリケーションのソースコードをユーザーと共有することに問題がなければ、[[https://en.wikipedia.org/wiki/GNU_Affero_General_Public_License][AGPL ライセンス]] の条件を受け入れることもできます。_

ソース位置情報が利用可能であれば、最小限の手間で他の入力ファイルフォーマットや他の YAML ライブラリも同様に使用できます。そのためには、AST を [[https://github.com/ahrefs/atd/blob/master/atd-jsonlike/src/AST.mli][JSON ライクな AST]] に変換する [[https://github.com/ahrefs/atd/blob/master/atd-yamlx/src/Atd_yamlx.ml][ブリッジ]] が必要です。
