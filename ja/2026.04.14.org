* YAMLx: 純粋 OCaml 製の YAML 1.2/1.1 ライブラリ
:PROPERTIES:
:CUSTOM_ID: 3
:END:
Archive: https://discuss.ocaml.org/t/ann-yamlx-pure-ocaml-yaml-1-2-1-1-library/17970/1

** Martin Jambon によるアナウンス


こんにちは、

YAML 1.2 および 1.1 標準の完全な実装である [[https://github.com/mjambon/yamlx][YAMLx]] の公開をお知らせできることを嬉しく思います。新しいアプリケーションでの設定ファイル形式としての利用や、他のアプリケーションの YAML ファイルの解析など、YAML の主要な用途のほとんどをサポートすることを目指しています。

詳しい説明については、[[https://github.com/mjambon/yamlx/releases/tag/0.1.0][リリースノート]]をご覧ください。

~opam install yamlx~ で始められます。その後、~yamlx~ コンパニオンコマンドを使って、お気に入りの YAML ファイルを正しくパース・解釈できるか確認してください。

ライブラリを使って YAML を JSON に変換する例を示します：

#+begin_src ocaml
(* Read YAML from stdin with YAMLx, print JSON to stdout with Yojson

   This is a demo showing how to use the YAMLx library with the default
   settings. 'YAMLx.Values.one_of_yaml_exn' offers options to restrict
   the input language or to force the interpretation as YAML 1.1 instead of
   YAML 1.2.

   Compile with:
     ocamlfind opt -o yaml-to-json \
       -package yamlx,yojson -linkpkg yaml_to_json.ml
*)

open Printf

let rec yojson_of_yamlx (x : YAMLx.value) : Yojson.Safe.t =
  match x with
  | Null _ -> `Null
  | Bool (_, x) -> `Bool x
  | Int (_, x) -> `Intlit (Int64.to_string x)
  | Float (_, x) -> `Float x
  | String (_, x) -> `String x
  | Seq (_, xs) -> `List (List.map yojson_of_yamlx xs)
  | Map (_, xs) ->
      `Assoc (List.map (fun (loc, k, v) ->
        match (k : YAMLx.value) with
        | String (_, k) -> (k, yojson_of_yamlx v)
        | _ ->
            ksprintf failwith "%s: only string keys are supported"
              (YAMLx.default_format_loc loc)
      ) xs)

let () =
  YAMLx.register_exception_printers ();
  stdin
  |> In_channel.input_all
  |> YAMLx.Values.one_of_yaml_exn
  |> yojson_of_yamlx
  |> Yojson.Safe.pretty_to_channel stdout
  |> print_newline
#+end_src

* ocaml-letsencrypt — OCaml で TLS 証明書を取得する方法
:PROPERTIES:
:CUSTOM_ID: 4
:END:
Archive: https://discuss.ocaml.org/t/ann-ocaml-letsencrypt-an-ocaml-way-to-get-tls-certificates/17975/1

** Calascibetta Romain によるアナウンス


[[https://github.com/robur-coop/ocaml-letsencrypt][ocaml-letsencrypt]]（バージョン 2.1.0）のリリースをお知らせできることを嬉しく思います。このバージョンでは、スケジューラや HTTP クライアントから独立して動作できる新しい API が導入されています。また、新しい [[https://github.com/robur-coop/jws][jws]] ライブラリを使った書き直しでもあり、[[https://github.com/ocaml-community/yojson][yojson]] の代わりに [[https://erratique.ch/logiciel/jsont][jsont]] ライブラリを利用するようになりました。

~ocaml-letsencrypt~ は、DNS、HTTP、ACME-TLS の3つの方法で証明書を取得するために必要な [[https://letsencrypt.org/fr/docs/challenge-types/][チャレンジ]] を実装しています。これにより、（例えば [[https://github.com/robur-coop/vif][Vif]] で実装された）ウェブサイトのデプロイ戦略を完全に OCaml で提供できるようになります。

私たちのコープ内では、以下のようなプロジェクトで活用しています：
- [[https://github.com/robur-coop/dns-letsencrypt-secondary/][~dns-letsencrypt-secondary~]]：セカンダリ DNS サーバとして機能し、特定ドメインの DNS チャレンジを実行して証明書と秘密鍵を生成できるユニカーネルです。一般的に、プライマリ DNS サーバである [[https://github.com/robur-coop/dns-primary-git][~primary-git~]] と連携して動作します。
- [[https://github.com/dinosaure/contruno][~contruno~]]：TLS リバースプロキシとして機能し、HTTP チャレンジを実行できるユニカーネルです。現在、[[https://discuss.ocaml.org/t/ann-mnet-a-new-tcp-ip-stack-for-unikernels-in-ocaml/17851][~mnet~]] を使って完全に書き直し中です。

ウェブサイトのデプロイがうまくいくことを祈っています。ハッピーハッキング！~jws~ も OCaml エコシステムの中でライブラリとして活躍できることを願っています。

* I18n 5.0 — あらゆる OCaml アプリを簡単に国際化
:PROPERTIES:
:CUSTOM_ID: 7
:END:
Archive: https://discuss.ocaml.org/t/ann-i18n-5-0-internationalisation-made-easy-for-any-ocaml-app/17981/1

** Vincent Balat によるアナウンス


Ocsigen チームは *ocsigen-i18n 5.0* のリリースをお知らせします。OCaml 向けの小さながら実用的な国際化ライブラリです。

このリリースの大きな変更点：ocsigen-i18n は Eliom への依存がなくなりました。これにより、*あらゆる OCaml プロジェクト* で使用できるようになりました。必要に応じて Tyxml や Eliom 向けのオプション拡張も利用できます。

ocsigen-i18n はもともと Be Sport の @sagotch によって書かれました。今回のリリースは [[https://github.com/mhjd][Habib]] の貢献によって実現し、[[https://www.irill.org][IRILL]] が資金を提供しました。

#+begin_quote
opam install ocsigen-i18n
#+end_quote

*** 仕組み

翻訳はシンプルな *TSV ファイル*（1行に1キー、1列に1言語）で記述します：

#+begin_example
foo                 This is a simple key.        Ceci est une clé toute simple.
a_human             a human                      un humain
bar                 I am {{x}}.                  Je suis {{x}}.
baz                 There {{{c?are||is an}}} apple{{{c?s||}}} here!  Il y a {{{c?des||une}}} pomme{{{c?s||}}} ici !
bu                  I am {{x %s}} ({{n %d}}).    Je suis {{x %s}} ({{n %d}}).
#+end_example

ミニテンプレート言語は以下をサポートしています：

- ~{{x}}~ 文字列変数 ~~x~
- ~{{x %d}}~ 指定したフォーマット指定子を使った型付き変数
- ~{{{c?yes||no}}}~ 2つの文字列を切り替えるオプションの真偽値 ~?c~

コード中では *PPX 拡張* が ~[%i18n key]~ を適切な呼び出しに変換します：

#+begin_src ocaml
print_endline [%i18n foo];
print_endline [%i18n bar ~x:[%i18n a_human]];
print_endline [%i18n baz ~c:(nb > 1)];
print_endline [%i18n bu ~x:"Jean-Michel" ~n:42];

(* 言語を明示的に切り替える *)
print_endline [%i18n foo ~lang:My_i18n.Fr];
#+end_src

現在の言語はミュータブルな参照で保持され、自由に切り替えられます（セッション/タブごとの言語が必要な場合は Eliom スコープ付き参照に置き換えることもできます）。

*** Dune との統合

このツールは Dune に自然に組み込めます。TSV ファイルから OCaml モジュールをルールで生成します：

#+begin_src lisp
(rule
 (target example_i18n.ml)
 (deps example_i18n.tsv)
 (action
  (run %{bin:ocsigen-i18n} --languages en,fr --input-file %{deps}
                           --output-file %{target})))
#+end_src

ライブラリ/実行ファイルに PPX を組み込みます：

#+begin_src lisp
(preprocess (pps ocsigen-i18n -- --default-module Example_i18n))
#+end_src

純粋な OCaml プロジェクトに必要なのはこれだけです。

*** Tyxml サポート

ジェネレータに ~--tyxml~ を渡すと、同じ ~[%i18n key]~ 式が文字列ではなく *Tyxml HTML 要素のリスト* を生成するようになります：

#+begin_src ocaml
(* HTML フラグメントを構築し、Tyxml ツリーにそのまま挿入できる *)
let greeting = [%i18n bar ~x:[%i18n a_human]]
#+end_src

変数自体も HTML ノードのリストにできるので、翻訳テキストとマークアップを自然に混在させられます：

#+begin_src ocaml
[%i18n bar ~x:[ txt "Jean-Michel ("
              ; txt (string_of_int id)
              ; txt ")" ]]
#+end_src

Tyxml モードで（属性などに）プレーンな文字列が必要な場合は、~S.~ をプレフィックスとして付けるだけです：

#+begin_src ocaml
[%i18n S.bar ~x:[%i18n S.a_human]]   (* 文字列出力 *)
#+end_src

*** Eliom サポート

クライアント–サーバー構成の Eliom アプリには ~--eliom~ を渡します。ジェネレータは ~.eliom~ ファイルを出力し（同じ翻訳を両側で利用可能）、~--tyxml~ を含意し、言語型に ~[@@deriving json]~ を追加してワイヤー越しにシリアライズできるようにします：

#+begin_src lisp
(rule
 (target example_i18n.eliom)
 (deps example_i18n.tsv)
 (action
  (run %{bin:ocsigen-i18n} --eliom --languages en,fr --input-file %{deps}
                           --output-file %{target})))
#+end_src

*** 複数の TSV ファイル

翻訳を複数のファイルに分割できます。PPX はモジュールパスを使って適切なものを見つけます：

#+begin_src ocaml
[%i18n foo]                          (* デフォルトモジュール *)
[%i18n MyI18n.foo]                   (* MyI18n.foo *)
[%i18n MyI18n.S.bar ~x:[%i18n S.foo]]
#+end_src

オプションの ~--prefix~ / ~--suffix~ フラグを使うと呼び出しサイトを簡潔に保てます（~[%i18n Feature.foo]~ → ~Pr_Feature_i18n.foo ()~）。

* ATD 4.1.0 — YAML サポートが追加されました
:PROPERTIES:
:CUSTOM_ID: 8
:END:
Archive: https://discuss.ocaml.org/t/ann-atd-4-1-0-now-with-yaml-support/17982/1

** Martin Jambon によるアナウンス


[[https://github.com/ahrefs/atd][ATD]] に、YAML ファイルやその他のフォーマットから最小限の手間でデータを読み込む OCaml サポートが追加されました。具体的には：

1. [[https://github.com/ahrefs/atd/blob/master/atd-yamlx/examples/app_config.atd][設定データの型を定義します]]。
2. ~atdml --mode jsonlike app_config.atd~ が、ソース位置情報を含む JSON ライクな AST を設定データ構造に変換する OCaml コードを生成します。
3. [[https://github.com/ahrefs/atd/blob/master/atd-yamlx/examples/main.ml][プログラム]] が YAML ファイルを読み込み、JSON ライクなツリーに変換し、生成された関数を呼び出して OCaml データ構造に変換します。
4. 設定が不正または不完全な場合、アプリケーションが有用なエラーメッセージを出力します。

[[https://github.com/ahrefs/atd/blob/master/atd-yamlx/examples][サンプル]] がディストリビューションに含まれています。[[https://discuss.ocaml.org/t/ann-yamlx-pure-ocaml-yaml-1-2-1-1-library/17970][最近リリースされた YAMLx ライブラリ]] を使用しています。

_YAMLx は、行われた作業に対する資金提供と [[https://github.com/mjambon/yamlx/blob/main/FUNDING.md][全員への許可的な ISC ライセンスの解放]] のために、寛大なスポンサーを必要としています。または、アプリケーションのソースコードをユーザーと共有することに問題がなければ、[[https://en.wikipedia.org/wiki/GNU_Affero_General_Public_License][AGPL ライセンス]] の条件を受け入れることもできます。_

ソース位置情報が利用可能であれば、他の入力ファイルフォーマットや他の YAML ライブラリも最小限の手間で同様に使用できます。そのためには、AST を [[https://github.com/ahrefs/atd/blob/master/atd-jsonlike/src/AST.mli][JSON ライクな AST]] に変換する [[https://github.com/ahrefs/atd/blob/master/atd-yamlx/src/Atd_yamlx.ml][ブリッジ]] が必要です。
