* YAMLx：純粋 OCaml 実装の YAML 1.2/1.1 ライブラリ
:PROPERTIES:
:CUSTOM_ID: 3
:END:
Archive: https://discuss.ocaml.org/t/ann-yamlx-pure-ocaml-yaml-1-2-1-1-library/17970/1

** Martin Jambon さんによるアナウンス


こんにちは、

YAML 1.2 および 1.1 規格の完全な実装である [[https://github.com/mjambon/yamlx][YAMLx]] の公開をお知らせできることを嬉しく思います。新規アプリケーションの設定ファイル形式としての利用や、他のアプリケーションの YAML ファイルの解析など、YAML のほとんどのユースケースをサポートすることを目的としています。

詳細については [[https://github.com/mjambon/yamlx/releases/tag/0.1.0][リリースノート]] をご覧ください。

~opam install yamlx~ で始められます。次に ~yamlx~ コマンドを使って、お気に入りの YAML ファイルが正しく解析・解釈されるか確認してみてください。

以下は、ライブラリを使って YAML を JSON に変換する例です：

#+begin_src ocaml
(* Read YAML from stdin with YAMLx, print JSON to stdout with Yojson

   This is a demo showing how to use the YAMLx library with the default
   settings. 'YAMLx.Values.one_of_yaml_exn' offers options to restrict
   the input language or to force the interpretation as YAML 1.1 instead of
   YAML 1.2.

   Compile with:
     ocamlfind opt -o yaml-to-json \
       -package yamlx,yojson -linkpkg yaml_to_json.ml
*)

open Printf

let rec yojson_of_yamlx (x : YAMLx.value) : Yojson.Safe.t =
  match x with
  | Null _ -> `Null
  | Bool (_, x) -> `Bool x
  | Int (_, x) -> `Intlit (Int64.to_string x)
  | Float (_, x) -> `Float x
  | String (_, x) -> `String x
  | Seq (_, xs) -> `List (List.map yojson_of_yamlx xs)
  | Map (_, xs) ->
      `Assoc (List.map (fun (loc, k, v) ->
        match (k : YAMLx.value) with
        | String (_, k) -> (k, yojson_of_yamlx v)
        | _ ->
            ksprintf failwith "%s: only string keys are supported"
              (YAMLx.default_format_loc loc)
      ) xs)

let () =
  YAMLx.register_exception_printers ();
  stdin
  |> In_channel.input_all
  |> YAMLx.Values.one_of_yaml_exn
  |> yojson_of_yamlx
  |> Yojson.Safe.pretty_to_channel stdout
  |> print_newline
#+end_src

*** 資金調達について

YAMLx は現在 *AGPL* ライセンスでリリースされています。現在クラウドファンディングを実施中で、目標金額に達した時点でライセンスを全ユーザー向けの許容的な *ISC* ライセンスに切り替える予定です。一定額以上の支援者には即時 *商用ライセンス* が付与されます。詳細は [[https://github.com/sponsors/mjambon][GitHub Sponsors プロフィール]] をご覧ください。
      



* ocaml-letsencrypt：TLS 証明書を取得するための OCaml の方法
:PROPERTIES:
:CUSTOM_ID: 4
:END:
Archive: https://discuss.ocaml.org/t/ann-ocaml-letsencrypt-an-ocaml-way-to-get-tls-certificates/17975/1

** Calascibetta Romain さんによるアナウンス


[[https://github.com/robur-coop/ocaml-letsencrypt][ocaml-letsencrypt]]（バージョン 2.1.0）のリリースをお知らせできることを嬉しく思います。このバージョンでは、スケジューラや HTTP クライアントに依存せずに動作できる新しい API が導入されました。また、[[https://erratique.ch/logiciel/jsont][jsont]] ライブラリ（[[https://github.com/ocaml-community/yojson][yojson]] の代わりに）を利用する新しい [[https://github.com/robur-coop/jws][jws]] ライブラリを使って書き直されています。

~ocaml-letsencrypt~ は、DNS、HTTP、ACME-TLS の3つの方式で証明書を取得するために必要な [[https://letsencrypt.org/fr/docs/challenge-types/][チャレンジ]] を実装しています。これにより、ウェブサイトのデプロイ戦略（例えば [[https://github.com/robur-coop/vif][Vif]] を使って実装されたもの）を完全に OCaml で提供できます。

私たちの協同組合では以下のようなプロジェクトで使用しています：
- [[https://github.com/robur-coop/dns-letsencrypt-secondary/][~dns-letsencrypt-secondary~]]：セカンダリ DNS サーバーとして動作し、特定ドメインの DNS チャレンジを実行して証明書と秘密鍵を生成できる unikernel です。通常、プライマリ DNS サーバーである [[https://github.com/robur-coop/dns-primary-git][~primary-git~]] と連携して動作します。
- [[https://github.com/dinosaure/contruno][~contruno~]]：TLS リバースプロキシとして動作し、HTTP チャレンジを実行できる unikernel です。現在 [[https://discuss.ocaml.org/t/ann-mnet-a-new-tcp-ip-stack-for-unikernels-in-ocaml/17851][~mnet~]] を使って完全に書き直されています。

ウェブサイトのデプロイがうまくいくことを願っています。ハッピーハッキング！~jws~ も OCaml エコシステムの中でライブラリとして定着してくれることを期待しています。

* I18n 5.0 — あらゆる OCaml アプリを簡単に国際化
:PROPERTIES:
:CUSTOM_ID: 7
:END:
Archive: https://discuss.ocaml.org/t/ann-i18n-5-0-internationalisation-made-easy-for-any-ocaml-app/17981/1

** Vincent Balat さんによるアナウンス


Ocsigen チームは、OCaml 向けの小さくも実用的な国際化ライブラリ *ocsigen-i18n 5.0* のリリースをお知らせします。

今回のリリースにおける大きな変更点：ocsigen-i18n が Eliom に依存しなくなりました。*あらゆる OCaml プロジェクト*で使用できるようになり、必要に応じて Tyxml および Eliom 向けのオプション拡張も利用できます。

ocsigen-i18n はもともと Be Sport の @sagotch によって書かれました。今回のリリースは [[https://www.irill.org][IRILL]] の資金援助のもとで [[https://github.com/mhjd][Habib]] の作業によって実現しました。

#+begin_quote
opam install ocsigen-i18n
#+end_quote

*** 使い方

翻訳はシンプルな *TSV ファイル*に記述します（1行に1キー、1列に1言語）：

#+begin_example
foo                 This is a simple key.        Ceci est une clé toute simple.
a_human             a human                      un humain
bar                 I am {{x}}.                  Je suis {{x}}.
baz                 There {{{c?are||is an}}} apple{{{c?s||}}} here!  Il y a {{{c?des||une}}} pomme{{{c?s||}}} ici !
bu                  I am {{x %s}} ({{n %d}}).    Je suis {{x %s}} ({{n %d}}).
#+end_example

ミニテンプレート言語は以下をサポートしています：

- ~{{x}}~ 文字列変数 ~~x~
- ~{{x %d}}~ 指定フォーマット指定子を使った型付き変数
- ~{{{c?yes||no}}}~ 2つの文字列を切り替えるオプションのブール値 ~?c~

コード中では、*PPX 拡張* が ~[%i18n key]~ を適切な呼び出しに変換します：

#+begin_src ocaml
print_endline [%i18n foo];
print_endline [%i18n bar ~x:[%i18n a_human]];
print_endline [%i18n baz ~c:(nb > 1)];
print_endline [%i18n bu ~x:"Jean-Michel" ~n:42];

(* 言語を明示的に切り替える *)
print_endline [%i18n foo ~lang:My_i18n.Fr];
#+end_src

現在の言語はミュータブルな参照に保持されており、変更できます（セッション/タブごとの言語が必要な場合は Eliom のスコープ付き参照に置き換えることも可能です）。

*** Dune との統合

このツールは Dune に非常に自然な形で組み込めます。TSV ファイルから OCaml モジュールを生成するルールを記述します：

#+begin_src lisp
(rule
 (target example_i18n.ml)
 (deps example_i18n.tsv)
 (action
  (run %{bin:ocsigen-i18n} --languages en,fr --input-file %{deps}
                           --output-file %{target})))
#+end_src

ライブラリ/実行ファイルに PPX を組み込みます：

#+begin_src lisp
(preprocess (pps ocsigen-i18n -- --default-module Example_i18n))
#+end_src

純粋な OCaml プロジェクトに必要なのはこれだけです。

*** Tyxml サポート

ジェネレーターに ~--tyxml~ を渡すと、同じ ~[%i18n key]~ 式が文字列の代わりに *Tyxml HTML 要素のリスト* を生成するようになります：

#+begin_src ocaml
(* HTML フラグメントを構築し、Tyxml ツリーに挿入可能 *)
let greeting = [%i18n bar ~x:[%i18n a_human]]
#+end_src

変数自体も HTML ノードのリストにできるため、翻訳されたテキストとマークアップを自然に組み合わせられます：

#+begin_src ocaml
[%i18n bar ~x:[ txt "Jean-Michel ("
              ; txt (string_of_int id)
              ; txt ")" ]]
#+end_src

Tyxml モードで（例えば属性に使う場合など）プレーンな文字列が必要な場合は、~S.~ を前置するだけです：

#+begin_src ocaml
[%i18n S.bar ~x:[%i18n S.a_human]]   (* 文字列出力 *)
#+end_src

*** Eliom サポート

クライアント・サーバー構成の Eliom アプリでは、~--eliom~ を渡します。ジェネレーターは ~.eliom~ ファイルを生成し（同じ翻訳が両サイドで利用可能になります）、~--tyxml~ が暗黙的に有効になります。また、言語タイプに ~[@@deriving json]~ が付加されるため、ネットワーク越しにシリアライズできます：

#+begin_src lisp
(rule
 (target example_i18n.eliom)
 (deps example_i18n.tsv)
 (action
  (run %{bin:ocsigen-i18n} --eliom --languages en,fr --input-file %{deps}
                           --output-file %{target})))
#+end_src

*** 複数の TSV ファイル

翻訳を複数のファイルに分割できます。PPX はモジュールパスを使って適切なファイルを見つけます：

#+begin_src ocaml
[%i18n foo]                          (* デフォルトモジュール *)
[%i18n MyI18n.foo]                   (* MyI18n.foo *)
[%i18n MyI18n.S.bar ~x:[%i18n S.foo]]
#+end_src

オプションの ~--prefix~ / ~--suffix~ フラグを使えば、呼び出し箇所を簡潔に保てます（~[%i18n Feature.foo]~ → ~Pr_Feature_i18n.foo ()~）。

翻訳を楽しんでください！
      



* ATD 4.1.0 — YAML サポートが追加されました
:PROPERTIES:
:CUSTOM_ID: 8
:END:
Archive: https://discuss.ocaml.org/t/ann-atd-4-1-0-now-with-yaml-support/17982/1

** Martin Jambon さんによるアナウンス


[[https://github.com/ahrefs/atd][ATD]] に、YAML ファイルやその他のフォーマットから最小限の手間でデータを読み込む OCaml サポートが追加されました。具体的には次のような流れになります：

1. [[https://github.com/ahrefs/atd/blob/master/atd-yamlx/examples/app_config.atd][設定データの型を定義する]]。
2. ~atdml --mode jsonlike app_config.atd~ で、ソース位置情報を含む JSON ライクな AST を設定データ構造に変換する OCaml コードを生成する。
3. [[https://github.com/ahrefs/atd/blob/master/atd-yamlx/examples/main.ml][プログラムが]] YAML ファイルを読み込み、JSON ライクなツリーに変換し、生成された関数を呼び出して OCaml データ構造に変換する。
4. 設定が不正または不完全な場合に、アプリケーションが有用なエラーメッセージを出力する。

配布物には [[https://github.com/ahrefs/atd/blob/master/atd-yamlx/examples][サンプル]] が含まれています。[[https://discuss.ocaml.org/t/ann-yamlx-pure-ocaml-yaml-1-2-1-1-library/17970][最近リリースされた YAMLx ライブラリ]] を使用しています。

_YAMLx がこれまでの作業への資金援助と [[https://github.com/mjambon/yamlx/blob/main/FUNDING.md][全ユーザーへの許容的な ISC ライセンスの解放]] のために、寛大なスポンサーを必要としていることに注意してください。あるいは、アプリケーションのソースコードをユーザーと共有することに問題がなければ [[https://en.wikipedia.org/wiki/GNU_Affero_General_Public_License][AGPL ライセンス]] の条件を受け入れることもできます。_

ソース位置情報が利用可能であれば、他の入力ファイル形式や他の YAML ライブラリも最小限の手間で同様に使用できます。そのためには、AST を [[https://github.com/ahrefs/atd/blob/master/atd-jsonlike/src/AST.mli][JSON ライクな AST]] に変換する [[https://github.com/ahrefs/atd/blob/master/atd-yamlx/src/Atd_yamlx.ml][ブリッジ]] が必要です。
